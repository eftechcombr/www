# AGENTS.md

## Dev environment tips
- Install [Hugo](https://gohugo.io/installation/) v0.163.3+ (extended edition) before running builds.
- Use `hugo server -D` for the local dev server with hot reload (includes drafts).
- Run `hugo --gc --minify` for production builds — output goes to `public/`.
- Clone with `git clone --recurse-submodules` to include the Blowfish theme.
- Store generated artefacts in `.context/` so reruns stay deterministic.

## Testing instructions
- Run `hugo --gc --minify` to verify the site builds without errors.
- Use `hugo server -D` for visual review while iterating on content or config.
- The CI pipeline (`hugo --gc --minify` + lychee link check) runs automatically on PRs.
- Verify internal links use trailing slashes (e.g., `/contato/`) to pass the link checker.

## PR instructions
- Follow Conventional Commits (for example, `feat(content): add new service page`).
- Cross-link new docs in `.context/docs/README.md` and `.context/agents/README.md` so future agents can find them.
- Ensure `hugo --gc --minify` succeeds locally before opening a PR.
- Verify no broken links — lychee runs in CI and will block merges on failures.

## Repository map
- `archetypes/` — Hugo content scaffolding templates. Edit when front matter schema changes.
- `assets/` — Images for Hugo Pipes processing (logo). Edit when adding optimised images.
- `config/` — Hugo site configuration (TOML). Edit to change site behaviour, theme params, menus, languages.
- `content/` — Markdown content in Portuguese (pt-br). Edit to add/update pages, blog posts, services.
- `data/` — Hugo data files for templates (currently empty). Edit when structured data is needed in templates.
- `i18n/` — Translation files (currently empty, theme defaults used). Edit to override Blowfish translations.
- `layouts/` — Custom layout overrides for the Blowfish theme (currently empty). Edit to customise templates without modifying the theme.
- `public/` — Build output directory (generated, gitignored). Do not edit — generated by `hugo --gc --minify`.
